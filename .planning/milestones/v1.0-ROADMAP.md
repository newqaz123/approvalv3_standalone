# Milestone v1.0: Core Approval Workflow System

**Status:** ✅ SHIPPED 2026-02-13
**Phases:** 1-8 (including 1.1, 1.2, 4.1)
**Total Plans:** 63

## Overview

This milestone delivers an internal approval workflow system for ~30 users that replaces email-based tracking with centralized visibility. The journey begins with authentication and user management, builds core request submission workflow, implements sophisticated approval routing engine with any-one-per-level logic, adds engineering solution workflow, provides dashboard visibility and audit trails, and completes with admin configuration tools for maintaining approval hierarchies.

## Milestone Summary

**Decimal Phases:**

- Phase 1.1: Fix Critical Authentication & Security Gaps (inserted after Phase 1 for urgent security fixes)
- Phase 1.2: Fix Data Consistency & User Management Issues (inserted after Phase 1.1 for data consistency)
- Phase 4.1: Auto Refresh Overlay Window (inserted after Phase 4 for UAT-identified gaps)

**Key Decisions:**

- Level-based approval with any-one-per-level (Rationale: Departments have multiple users at same rank)
- Manual completion after SendBackToRequester (Rationale: Integration point with E-ordering system)
- Admin-only workflow configuration (Rationale: Prevents accidental changes to approval chains)
- Local file storage instead of S3 (Rationale: User-approved for easier VPS deployment)
- JWT metadata in middleware + database checks in Server Actions (Rationale: Edge runtime compatibility + defense-in-depth)
- RequestActivity.requestId optional (Rationale: Single source of truth for audit trail without separate system_activities table)

**Issues Resolved:**

- Fixed webhook doesn't create Prisma records (Phase 1.1)
- Fixed role-based access bypass (Phase 1.1)
- No transaction rollback (Phase 1.2)
- Incorrect email invitation method (Phase 1.2)
- Incomplete edit functionality (Phase 8)
- Foreign key violation in user update (Phase 8)

**Technical Debt Incurred:**

- Temporary middleware bypass for metadata refresh (?refresh=true) - needs proper solution before production
- Edit department functionality not fully wired - deferred to future iteration
- Local file storage needs backup strategy in production
- Console.log statements in drag-and-drop handlers should be removed before production
