# Notification System UAT (User Acceptance Test)

**Version:** 1.0
**Date:** 2026-02-22
**Feature:** Stage-Based Notification System (Quick Task 027)
**Test Type:** Manual Verification

---

## Overview

This UAT document verifies that all 6 notification triggers work correctly and reach the intended recipients. Each test scenario includes prerequisites, step-by-step instructions, and expected results.

---

## Prerequisites

### Required Test Users

Create or identify these test users before testing:

| Role | Department | Purpose |
|------|------------|---------|
| **Requester A** | Any non-Engineering dept | Creates requests |
| **Approver A** | Same dept as Requester A | Approves at department level |
| **Engineer 1** | Engineering | Receives engineering notifications |
| **Engineer 2** | Engineering | Verifies all engineers get notified |
| **Engineer 3** | Engineering | Verifies all engineers get notified |
| **Dept User 1** | Same dept as Requester A | Verifies department-wide notifications |
| **Dept User 2** | Same dept as Requester A | Verifies department-wide notifications |

### Test Data Setup

1. Ensure Engineering department exists with `type: 'ENGINEERING'`
2. Ensure at least one non-Engineering department has multiple users
3. Verify notification bell icon appears in navbar (top right)
4. Clear existing notifications for clean test results

## Test Scenarios

### TEST 1: Improvement Request Rejected

**Trigger:** Approver rejects request during ImprovementRequest stage

**Expected Recipients:** Requester ONLY

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Login as **Requester A** | - |
| 2 | Create a new request titled "UAT Test Request 1" | Request created successfully |
| 3 | Note the request ID | - |
| 4 | Logout and login as **Approver A** | - |
| 5 | Navigate to the request detail modal | Modal opens |
| 6 | Click **Reject** button | Reject dialog appears |
| 7 | Enter rejection reason: "UAT Test - Insufficient details" | - |
| 8 | Confirm rejection | Toast shows success |
| 9 | Logout and login as **Requester A** | - |
| 10 | Click notification bell icon | Notification popover opens |
| 11 | Verify notification appears | ✅ Title: "Request Rejected" |
| 12 | Verify message content | ✅ Contains: "❌ Request Rejected" and rejection reason |
| 13 | Click notification | ✅ Opens request detail modal |

**Verification Checklist:**

- [ ] Requester receives notification
- [ ] Notification shows rejection reason
- [ ] Clicking notification opens correct request
- [ ] Other users (Engineers, Dept Users) did NOT receive this notification
