  <name>Create Docker Compose Configuration</name>
  <files>docker-compose.yml</files>
  <action>
    Create `docker-compose.yml` with services:
    
    1. `db`:
       - Image: postgres:15-alpine
       - Environment: POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB
       - Volumes: db_data:/var/lib/postgresql/data
       - Healthcheck: pg_isready
       - Logging: driver: "json-file", options: { max-size: "10m", max-file: "3" }
       - Deploy: resources: limits: { cpus: "1", memory: "512M" }
       
    2. `migrate`:
       - Build: target `builder` (has deps)
       - Command: `npx prisma migrate deploy`
       - Depends_on: db (healthy)
       - Environment: DATABASE_URL
       - Logging: driver: "json-file", options: { max-size: "10m", max-file: "3" }
       
    3. `app`:
       - Build: target `runner` (minimal)
       - Ports: "3000:3000"
       - Environment: from .env file
       - Volumes: uploads_data:/app/uploads
       - Depends_on: db (healthy), migrate (completed)
       - Restart: always
       - Logging: driver: "json-file", options: { max-size: "10m", max-file: "3" }
       - Deploy: resources: limits: { cpus: "1", memory: "1G" }
       
    Define volumes `db_data` and `uploads_data`.
  </action>
  <verify>
    docker compose config
  </verify>
  <done>
    Compose configuration is syntactically valid and defines all 3 services with resource limits and logging config.
  </done>
