<task type="auto">
  <name>Task 2: Implement Advanced Update Logic (Role, Name & Email)</name>
  <files>src/server-actions/users.ts</files>
  <action>
    Update `updateUser` in `src/server-actions/users.ts`:
    
    1. **Role Handling**: Remove the forced auto-role logic. Use `input.role` if provided, falling back to auto-role only if undefined.
    
     2. **Email Synchronization**:
        - Check if `input.email` differs from `existingUser.email`.
        - If changed:
          - Call `clerkClient.emailAddresses.create({ userId, emailAddress: input.email, verified: true, primary: true })`.
          - (Optional) Delete old email address from Clerk to keep the account clean.
    
     3. **Name & Metadata Sync**:
        - Split `input.name` into `firstName` (first word) and `lastName` (rest of string).
        - Call `clerkClient.users.updateUser(userId, { firstName: ..., lastName: ..., publicMetadata: { ... } })`.
        - Ensure `publicMetadata` includes the final `role` and `departmentId`.
    
     4. **Audit Logging**:
        - Log all admin changes to `RequestActivity` with `action='user_admin_changed'` and `requestId='SYSTEM'`.
        - Record: changed field (role, department, level, email, name), old value, new value.
    
     5. **Prisma Update**: Update `email`, `role`, `departmentId`, `name` in Prisma.
  </action>
  <verify>
    `updateUser` function contains logic for `clerkClient.users.updateUser` (updating name/metadata) and `emailAddresses.create`.
  </verify>
  <done>
    Server action handles role overrides, name updates, and email changes via Clerk API.
  </done>
</task>

  <files>src/server-actions/users.ts</files>
  <action>
    Add error handling and rollback to `updateUser`:
    
    - Wrap the Prisma update in a `try/catch` block.
    - In the `catch` block (Prisma failed):
      - Revert Clerk changes:
        - Restore original `firstName`, `lastName`, `publicMetadata` using `clerkClient.users.updateUser`.
        - If email was changed:

          - Delete the new (failed) email.
     - Throw a descriptive error to the UI.
   
   - **Session Refresh Note**: If role is changed, return a message: "User must re-login for role change to take effect."
  </action>
  <verify>
    `updateUser` has a `catch` block that calls `clerkClient.users.updateUser` or `emailAddresses` to revert changes.
  </verify>
  <done>

  </done>
</task>
