Implement three query functions:

1. **getAuditTrailForRequest(requestId: string)**
   - Queries all RequestActivity records for a specific request
   - Includes user (name, email) for display
   - Includes request (title, department) for context
   - Ordered by createdAt DESC (newest first)
   - Admin-only access (requireAdmin())
   - Use case: Activity timeline, request detail view

2. **getAuditTrailForDateRange(startDate: Date, endDate: Date, options?)**
   - Queries RequestActivity within date range
   - Includes user and request data
   - Options: pagination (skip, take) for large result sets
   - Admin-only access (requireAdmin())
   - Use case: Admin audit log viewing, export prep

3. **getAuditTrailForUser(userId: string, options?)**
   - Queries all activities by a specific user
   - Includes request context for each activity
   - Options: pagination, date range filter
   - Access control: Admins see all, users see own activities only
   - Use case: User activity history, compliance reporting

Query pattern per RESEARCH.md:
```typescript
const activities = await prisma.requestActivity.findMany({
  where: {
    requestId: requestId,
    // OR date range filter
    createdAt: {
      gte: startDate,
      lte: endDate,
    }
  },
  include: {
    user: {
      select: {
        id: true,
        name: true,
        email: true,
      },
    },
    request: {
      select: {
        id: true,
        title: true,
        departmentId: true,
        status: true,
      },
    },
  },
  orderBy: {
    createdAt: 'desc',
  },
  // Pagination for large results
  take: options?.take || 100,
  skip: options?.skip || 0,
})
```

Note: Indexes already exist on requestId, userId, createdAt from schema (see lines 179-181 of schema.prisma).

Return types should match existing ActivityTimeline component interface where possible.
