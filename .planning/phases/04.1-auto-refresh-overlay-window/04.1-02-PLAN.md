1. **approveRequest function**:
   - Add optional parameter: `expectedUpdatedAt?: string | Date`
   - Before making any changes, fetch current request.updatedAt
   - Compare timestamps with 1-second tolerance:
     ```typescript
     const checkStale = (current: Date, expected: string | Date | undefined): boolean => {
       if (!expected) return false // No check if not provided (backward compatible)
       const currentTime = current.getTime()
       const expectedTime = new Date(expected).getTime()
       return Math.abs(currentTime - expectedTime) > 1000
     }
     ```
   - If stale, return: `{ success: false, stale: true, message: 'This request was updated by another user.' }`
   - If not stale, proceed with existing logic
   - Update return type to include `stale?: boolean`

2. **rejectRequest function**:
   - Same pattern as approveRequest
   - Add `expectedUpdatedAt` parameter
   - Check staleness before rejecting
   - Return stale flag if conflict detected

3. Add helper function at top of file:
   ```typescript
   function isStaleData(currentUpdatedAt: Date, expectedUpdatedAt?: string | Date): boolean {
     if (!expectedUpdatedAt) return false
     const diff = Math.abs(currentUpdatedAt.getTime() - new Date(expectedUpdatedAt).getTime())
     return diff > 1000 // 1-second tolerance for database precision
   }
   ```

Ensure backward compatibility - existing calls without expectedUpdatedAt should work unchanged.
