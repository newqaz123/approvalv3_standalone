## Decisions Made

- **150ms duration**: Provides subtle fade without feeling slow or distracting
- **ease-in timing**: Starts slow and accelerates, feels natural for content appearing
- **Animation on content area only**: Applied to inner content div, not Dialog shell, for better visual effect
- **State reset on load start**: contentVisible set to false when loading starts to re-trigger animation on refresh
- **No animation on skeleton**: Early return in loading state prevents fade-in on initial skeleton render
