  <action>
    Create src/server-actions/reports.ts with:

    1. 'use server' directive
    2. Import auth from @clerk/nextjs/server
    3. Import prisma from @/lib/prisma
    4. Import generateRequestPDF from @/lib/pdf
    5. Create in-memory rate limit Map: pdfRateLimit = new Map<string, { count: number; resetAt: number }>()
    6. Export async function exportRequestAsPDF(requestId: string)

    Function implementation:
    a. Get userId from auth() - return error if not authenticated
    b. Rate limit check (3 PDFs per minute):
       - Check existing entry in pdfRateLimit for userId
       - If within time window and count >= 3, return error
       - Otherwise increment count or create new entry
    c. Fetch request with comprehensive Prisma query including:
       - requester (select id, name, email)
       - department (select id, name)
       - fileAttachments (include uploadedBy, orderBy createdAt)
       - activities (include user, orderBy createdAt desc)
       - approvals (include approver, requiredApprover, orderBy requiredLevel)
       - solutions (include submittedBy, fileAttachments with uploadedBy, take: 1, orderBy createdAt desc)
    d. Validate request exists - return error if not found
    e. Validate status is FinalApproval or Completed - return error if not
    f. Validate all approvals have status 'approved' - return error if any pending
    g. Fetch current user name for metadata
    h. Determine completedAt from activities (filter for 'manually_completed' action or toStatus='Completed', sort by createdAt, take first)
    i. Build pdfData object matching RequestPDFData interface
    j. Call generateRequestPDF(pdfData) to get buffer
    k. Generate filename: sanitize title (remove special chars, limit to 50 chars), format date as YYYY-MM-DD, combine as Request_TOPIC_YYYY-MM-DD.pdf
    l. Return { success: true, data: base64 buffer, filename, contentType: 'application/pdf' }

    Error handling:
    - Wrap in try/catch
    - Return { success: false, error: message } on any error
  </action>
