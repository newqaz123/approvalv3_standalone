### Architectural Notes

**Defense-in-Depth Security Model:**
- **Layer 1 (Middleware):** Fast edge routing using JWT metadata (blocks obvious non-admins at CDN)
- **Layer 2 (Server Actions):** Database-backed authorization via `requireAdmin()` (real-time enforcement)

**Why JWT Metadata in Middleware?**
Next.js middleware runs at the **edge** (CDN locations), not Node.js runtime. Prisma requires Node.js, so database queries fail in middleware. JWT metadata is acceptable for routing because:
1. Performance: Edge routing with JWT is extremely fast
2. Defense-in-depth: Server Actions still enforce with database checks
3. Acceptable staleness: 60-second JWT cache is acceptable for routing layer
4. Real enforcement: Actual authorization happens in Server Actions

**Why This Is Secure:**
- Before: ANY authenticated user could access `/admin` routes (critical vulnerability)
- After: Non-admins redirected at middleware, blocked again at Server Actions
- Role changes: Take effect immediately in Server Actions (database queries), within 60s in middleware (JWT cache)
