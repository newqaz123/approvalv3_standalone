# Current State
The auth helpers and middleware currently have TEMPORARY bypasses:

**src/lib/auth.ts:**
- `requireAdmin()` has commented-out role check (lines 29-33)
- Returns userId for any authenticated user
- Uses stale JWT metadata instead of database

**src/middleware.ts:**
- Admin route protection has commented-out redirect (lines 25-30)
- Any authenticated user can access /admin routes
- Uses stale JWT metadata instead of database

**Root cause:** Clerk JWT tokens cache `publicMetadata` from token issuance (default 60 seconds). When admin changes a user's role, the user's JWT still has old role until token expires.

**Solution:** Query Prisma directly for current role (always reflects latest changes).

```typescript
/**
 * Verifies the current user is an admin using database-backed role check.
 * Returns null if not authenticated or not an admin.
 *
 * This approach bypasses stale JWT metadata by checking Prisma directly,
 * ensuring role changes take effect immediately.
 */
export async function requireAdmin() {
  const { userId } = await auth()
  if (!userId) {
    return null
  }

  // Check Prisma for current role (always reflects latest changes)
  const user = await prisma.user.findUnique({
    where: { id: userId },
    select: { role: true, isActive: true },
  })

  // Verify user exists, is active, and has admin role
  if (!user || !user.isActive || user.role !== 'admin') {
    return null
  }

  return userId
}
```

```typescript
  if (isProtectedRoute(req)) {
    const { redirectToSignIn } = await auth()

    if (!isAuthenticated) {

    }

})
```
