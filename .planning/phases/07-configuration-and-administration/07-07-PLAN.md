    1. Import AlertDialog components: `AlertDialog, AlertDialogAction, AlertDialogCancel, AlertDialogContent, AlertDialogDescription, AlertDialogFooter, AlertDialogHeader, AlertDialogTitle` from `@/components/ui/alert-dialog`.

    2. Add state to track a pending drag operation:
       ```typescript
       const [pendingMove, setPendingMove] = useState<{
         userId: string
         userName: string
         fromLevel: number
         toLevel: number
         isExternal: boolean
         previousState: Record<number, User[]>
       } | null>(null)
       ```

    3. Refactor `handleDragEnd`:
       - Keep the logic that determines userId, currentLevel, targetLevel, and user.

    4. Add a `confirmMove` async function that:
       - Reads `pendingMove` state
       - Calls the appropriate server action (updateHierarchy for external, updateUserLevel for internal) — same logic currently in handleDragEnd
       - On success: shows toast, clears `pendingMove`
       - On failure: reverts to `previousState` from `pendingMove`, shows error toast, clears `pendingMove`

    5. Add a `cancelMove` function that:
       - Reverts `usersByLevel` to `pendingMove.previousState`
       - Clears `pendingMove`

    6. Render an AlertDialog (outside the DndContext, controlled by `open={!!pendingMove}`):

       - Description: "Move {userName} from Level {fromLevel} to Level {toLevel}?"

       - Cancel button: calls `cancelMove`
       - Confirm button: calls `confirmMove`
       - `onOpenChange`: if closed via escape/overlay click, call `cancelMove`

    7. The optimistic UI update happens immediately on drag (so the user sees the card in its new position as a preview), but the server call only fires on confirm. Cancel reverts the card to its original position.
