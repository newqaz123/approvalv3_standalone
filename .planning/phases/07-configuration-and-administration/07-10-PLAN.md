<objective>
Fix two UAT-identified gaps in the hierarchy management feature:
1. Remove overly strict validation that forces hierarchies to start at Level 1 and have no gaps between levels. The approval engine (createApprovalChain in approvals.ts) already handles gaps gracefully by skipping levels with no approvers (line 88: `if (hasInternalUsers || hasExternalApprovers)`), so these validation rules block legitimate configurations unnecessarily.
2. Add External badge to hierarchy user cards for cross-department approvers. The data is already set correctly by getHierarchyData() (isExternal: true for DepartmentApprovers) but the rendering components drop it before reaching the card.

Purpose: Unblock admins from configuring legitimate department hierarchies and provide visual clarity for external approvers.
Output: Fixed validation logic and visible External badges on hierarchy user cards.
</objective>
