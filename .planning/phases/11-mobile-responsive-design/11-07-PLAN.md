<objective>
Fix admin mobile layout issues - remove double-container effect, fix nav overlap, increase text size

**Purpose:** Users reported admin dashboard shows large empty space on right side (desktop layout visible instead of mobile cards), top navigation bar overlaps content, and text is too small. These issues are caused by (1) admin pages using 'container mx-auto' wrapper creating double-container effect, (2) layout main padding pt-8 too small for 64px mobile nav, (3) AdminCard using text-sm (14px) below 16px mobile minimum.

**Output:** Admin pages display full-width mobile cards with readable 16px text, content clears 64px mobile nav bar
</objective>

**Root Cause (from .planning/debug/admin-mobile-layout-issues.md):**
1. Double-container effect: Admin pages use `<div className="container mx-auto py-6">` wrapper, but dashboard layout already applies `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
2. Nav overlap: Main content has `pt-8` (32px) but mobile nav is 64px tall
3. Small text: AdminCard detail rows use `text-sm` (14px) below 16px mobile minimum

**Debug Session:** .planning/debug/admin-mobile-layout-issues.md
