**UAT Gap (Test 11):**
- "Dialog appears but can't touch it - it disappears when touched. Dialog is not interactive on mobile."

**Root Cause (from .planning/debug/mobile-reject-dialog-not-interactive.md):**
Custom reject dialog in MobileApprovalActions (lines 141-197) is implemented as a regular div with z-[100], rendered within the RequestDrawer component's React tree. Although visually layered above the drawer (z-50), it remains within the drawer's event capture context. When users tap the dialog, the Vaul drawer's overlay receives the touch event and interprets it as a "click outside" gesture, closing the drawer and dismissing the dialog.

**Current implementation (broken):**
```tsx
{showRejectDialog && (
  <div className="fixed inset-0 z-[100] flex items-end justify-center">
    <div className="absolute inset-0 bg-black/40" onClick={handleCancelReject} />
    <div className="relative z-10 w-full max-w-lg bg-white rounded-t-[10px] p-4...">
      {/* Dialog content */}
    </div>
  </div>
)}
```

**Solution:** Use Radix AlertDialog which uses AlertDialogPortal to render content at document.body level, completely escaping the parent drawer's stacking context and event capture. Reference: src/components/requests/cancel-request-dialog.tsx (working example).

**Debug Session:** .planning/debug/mobile-reject-dialog-not-interactive.md
</context>
