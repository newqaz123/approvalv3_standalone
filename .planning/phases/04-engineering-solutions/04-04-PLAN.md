  <name>Task 1: Add engineering-specific query functions</name>
  <files>src/server-actions/requests.ts</files>
  <action>
    Extend `src/server-actions/requests.ts` with engineering functions:

    1. `getRequestsNeedingEngineeringAction(userId: string)`:
       Returns two categories:

       a. Requests needing solution submission:
          - Status = SentToEngineer
          - No solution submitted yet (or check Solution table)
          - Include: request details, department, requester, assigned engineers

       b. Solutions needing user's approval:
          - Request status = DesignCostEstimationApproval
          - Has SolutionApproval record where:
            - For custom chain: requiredApproverId === userId AND status === pending AND all previous orders approved
            - For hierarchy: requiredLevel === user.level AND status === pending AND all previous orders approved
          - Include: request details, solution details, approval info

       ```typescript
       interface NeedsActionResult {
         needsSolution: Array<{
           request: Request & { department: Department; requester: User }
           assignedEngineers: User[]
         }>
         needsApproval: Array<{
           request: Request
           solution: Solution
           approval: SolutionApproval
         }>
       }
       ```

    2. `assignEngineers(requestId: string, engineerIds: string[])`:
       - Validate caller is engineering user
       - Validate request status is SentToEngineer
       - Delete existing assignments
       - Create new RequestEngineerAssignment records
       - Log activity (action: 'engineers_assigned')
       - revalidatePath('/engineering')
       - Return { success: true }

    3. `getEngineeringUsers()`:
       - Return all users in engineering department
       - Used for Person in Charge selector
       - Include: id, name, email, level
       - Order by level desc, name asc

    4. `getRequestsForEngineering(filters)`:
       - Base query: status IN (SentToEngineer, DesignCostEstimationApproval)
       - Filters: departmentId (origin), assignedEngineerId, status, dateRange, search
       - Include: request, department, requester, assignedEngineers, solution (if exists)
       - Used for engineering request list view
  </action>
  <verify>
    - TypeScript compiles: `npx tsc --noEmit`
    - Functions exported: getRequestsNeedingEngineeringAction, assignEngineers, getEngineeringUsers
    - Queries return correct data shape
  </verify>
