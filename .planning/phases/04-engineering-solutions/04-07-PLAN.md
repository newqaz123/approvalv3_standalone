<task type="auto">
  <name>Task 1: Create notification server actions</name>
  <files>src/server-actions/notifications.ts</files>
  <action>
    Create `src/server-actions/notifications.ts`:

    1. `getNotifications(userId: string, limit?: number)`:
       - Fetch notifications for user, ordered by createdAt desc
       - Include request relation (title only)
       - Default limit: 20
       - Return: Array<Notification & { request?: { title: string } }>

    2. `getUnreadCount(userId: string)`:
       - Count where userId && isRead === false
       - Return: number

    3. `markAsRead(notificationId: string)`:
       - Validate ownership (notification.userId === currentUser)
       - Update isRead: true, readAt: new Date()
       - revalidatePath('/')
       - Return { success: true }

    4. `markAllAsRead(userId: string)`:
       - Update all unread notifications for user
       - revalidatePath('/')
       - Return { success: true, count: number }

    5. `createNotification(data)`:
       - Helper function used by other server actions
       - Create notification record
       - Optionally trigger email notification

    6. `sendEmailNotification(notification, userEmail)`:
       - Skip email if SMTP not configured (check env vars)
       - Use nodemailer or Resend for email sending
       - Templates:
         - solution_ready: "Solution is ready for your request '{title}'"
         - final_approval_needed: "Your approval is needed for '{title}'"
         - approval_needed: "Your approval is needed for '{title}'"
       - Minimal email: subject + "Click to view: {link to request}"
       - Return { success: boolean, error?: string }

    7. Environment variables needed:
       - SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM
       - Or RESEND_API_KEY
       - Skip email if not configured (log warning)
  </action>
  <verify>
    - TypeScript compiles: `npx tsc --noEmit`
    - Functions exported: getNotifications, getUnreadCount, markAsRead, markAllAsRead
    - Queries return correct data shape
    - Email function handles missing config gracefully
  </verify>

  <action>
    Create `src/components/notifications/notification-bell.tsx`:

    1. Client component with state:
       - notifications: Array of notifications
       - unreadCount: number
       - isOpen: boolean (popover state)
       - isLoading: boolean

    2. Fetch on mount and on interval (optional: every 30 seconds):
       ```typescript
       useEffect(() => {
         async function fetchNotifications() {
           const [notifs, count] = await Promise.all([
             getNotifications(userId, 20),
             getUnreadCount(userId),
           ])
           setNotifications(notifs)
           setUnreadCount(count)
         }
         fetchNotifications()

         // Optional: refresh periodically
         const interval = setInterval(fetchNotifications, 30000)
         return () => clearInterval(interval)
       }, [userId])
       ```

    3. Bell icon with badge:
       ```tsx
       <Popover open={isOpen} onOpenChange={setIsOpen}>
         <PopoverTrigger asChild>
           <Button variant="ghost" size="icon" className="relative">
             <Bell className="h-5 w-5" />
             {unreadCount > 0 && (
               <span className="absolute -right-1 -top-1 flex h-5 w-5 items-center justify-center rounded-full bg-red-500 text-xs text-white">
                 {unreadCount > 9 ? '9+' : unreadCount}
               </span>
             )}
           </Button>
         </PopoverTrigger>
         <PopoverContent className="w-96 p-0" align="end">
           <NotificationList
             notifications={notifications}
             onMarkAsRead={handleMarkAsRead}
             onMarkAllAsRead={handleMarkAllAsRead}
           />
         </PopoverContent>
       </Popover>
       ```

    4. Handlers:
       - handleMarkAsRead: call markAsRead, update local state, decrement count
       - handleMarkAllAsRead: call markAllAsRead, update all local items, set count to 0

    5. Use shadcn/ui Popover, PopoverContent, PopoverTrigger, Button
    6. Use lucide-react Bell icon
  </action>
