## Gap Context

**Issue:** Engineering approvers cannot see Approve/Reject buttons in request detail modal (BLOCKER)

**Root Cause (from debug session):**
In request-detail-modal.tsx lines 118-126, the code loops through solutionApprovals and incorrectly passes `approval.id` to `canUserApproveSolution()`, but the function expects `solution.id`. The function queries `prisma.solution.findUnique()` with approval ID (wrong type), returns null, and sets canApproveSolution to false, so buttons never render.

**Current broken code (lines 118-126):**
```typescript
// Check if current user can approve this solution
for (const approval of approvals) {
  const checkResult = await canUserApproveSolution(approval.id)  // BUG: passing approval.id instead of solution.id
  if (checkResult.canApprove) {
    setCanApproveSolution(true)
    setSolutionApprovalId(approval.id)
    break
  }
}
```

**Function signature (solutions.ts line 449):**
```typescript
export async function canUserApproveSolution(solutionId: string) {
```

The function expects a solution ID but is receiving an approval ID, causing it to fail finding the solution and always returning `{ canApprove: false }`.
