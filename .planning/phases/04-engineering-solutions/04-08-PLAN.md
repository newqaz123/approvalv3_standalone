<objective>
Fix file upload authorization and "use server" export violation blocking solution submission

Purpose: TWO blockers prevent solution submission:

1. File upload authorization checks validate that current user is the original requester, but engineering users submitting solutions upload files to requests they didn't create. Two authorization checks block engineering users: (1) src/server-actions/files.ts:78 checks requesterId !== userId, (2) src/app/api/upload/route.ts:58 checks dbRequest.requesterId !== userId.

2. Next.js 'use server' directive only allows async function exports. The current solutions.ts file exports a Zod schema object (submitSolutionSchema) and TypeScript type (SubmitSolutionInput), which violates this rule and causes runtime error: "A 'use server' file can only export async functions, found object."

Output: Working solution submission with proper file upload authorization and file separation
</objective>
