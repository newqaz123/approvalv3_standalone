## Gap Context

**Issue:** Submit button visibility inconsistent - shows in /engineering dashboard but NOT in modal opened from /requests page (MAJOR)

**User-reported business logic clarification:**
"actually everyone in engineering department should see and submit solution!!"

**Root Cause (from debug session):**
request-detail-modal.tsx line 465 uses unreliable client-side `userRole === 'engineering'` check that fails when:
1. userRole state is undefined on first render (useState initializes before useEffect syncs from Clerk metadata)
2. Race condition between component mount and user metadata propagation
3. Modal is accessible from /requests page which has no server-side role protection

**Current broken code (line 465):**
```typescript
{!solution && request.status === 'SentToEngineer' && userRole === 'engineering' && (
```

**Why engineering dashboard works:**
- needs-action-list.tsx (line 156-160) shows button WITHOUT role check
- Engineering page has server-side role protection (page.tsx lines 31-33)
- Only engineering users can reach the dashboard, so button is safe to show

**Why modal doesn't work:**
- Modal can be opened from /requests page by any user
- Client-side userRole check fails due to async state sync timing
- Check fails even for legitimate engineering users

**Security boundary:**
- Button links to `/engineering/solutions/[requestId]`
- That page has server-side authorization (checks user.role === 'engineering')
- Non-engineering users clicking button get redirected to /dashboard
- Button visibility is informational UI, not security boundary
</context>
