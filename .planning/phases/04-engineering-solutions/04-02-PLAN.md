  <action>
    Create `src/components/solutions/solution-form.tsx`:

    1. Props interface:
       ```typescript
       interface SolutionFormProps {
         requestId: string
         requestTitle: string
         currentUserId: string
         engineeringUsers: Array<{ id: string; name: string; email: string; level: number | null }>
       }
       ```

    2. Use React Hook Form with Zod resolver:
       ```typescript
       const formSchema = z.object({
         title: z.string().min(1, 'Title is required').max(200),
         description: z.string().min(1, 'Description is required').max(5000),
         costEstimate: z.number({ invalid_type_error: 'Enter a valid cost' })
           .positive('Cost must be greater than 0')
           .min(0.01, 'Minimum cost is 0.01'),
         currency: z.enum(['THB']).default('THB'),
         timeline: z.string().max(200).optional().or(z.literal('')),
         conceptDesign: z.string().max(2000).optional().or(z.literal('')),
         useCustomApprovals: z.boolean().default(false),
         customApproverIds: z.array(z.string()).optional(),
       })
       ```

    3. Form state:
       - `showPreview: boolean` - toggle between form and preview
       - `attachedFiles: File[]` - files to upload with solution
       - Use React Hook Form's watch() for real-time validation feedback

    4. Form fields:
       - Title: Input (defaultValue: "Solution for: {requestTitle}")
       - Description: Textarea with character count
       - Cost Estimate: Input type="number" step="0.01" with `valueAsNumber: true`
       - Currency: Select (THB only for now, but extensible)
       - Timeline: Input (optional, placeholder: "e.g., 2-3 weeks")
       - Concept Design: Textarea (optional)
       - File Attachments: SolutionFileUpload component (PDF, images, CAD files, Office docs)
       - Approval routing toggle: Switch for "Use custom approval chain"
       - If custom: render CustomApprovalPicker

    5. Submit flow:
       a. handleSubmit validates form
       b. If valid and !showPreview, set showPreview = true
       c. If showPreview and confirmed:
          - Upload files first via /api/upload endpoint (returns uploaded file IDs)
          - Call submitSolution server action with uploaded file IDs array
          - Server action creates Solution record and associates FileAttachment records with solutionId using the file IDs
       d. On success, redirect to `/engineering` (engineering dashboard)
       e. On error, show toast with error message (sonner)

    FILE ATTACHMENT INTEGRATION:
    - The /api/upload endpoint creates FileAttachment records with null solutionId initially
    - When submitSolution is called, it receives the fileIds array and updates those FileAttachment records to set solutionId
    - This ensures files are properly linked to the Solution record in the database

    6. Use existing patterns from request-form.tsx for consistency (especially file upload handling)

    7. Link to view full request detail: "View original request" opens in new tab

       - Fetches list of engineering department users for approval picker
       - Returns 404 if request not found or wrong status
       - Returns 403 if user is not engineering

    2. Page layout:
       - Back link to engineering dashboard
       - Page title: "Submit Solution"
       - Request context card (title, description summary, department, requester name)
       - SolutionForm component with all required props

    3. Data fetching:
       ```typescript
       // Fetch request with validation
       const request = await prisma.request.findFirst({
         where: {
           id: requestId,
           status: 'SentToEngineer',
           isDeleted: false,
         },
         include: {
           requester: { select: { name: true, email: true } },
           department: { select: { name: true } },
         },
       })

       // Fetch engineering users for custom approval picker
       const engineeringDept = await prisma.department.findFirst({
         where: { type: 'ENGINEERING' },
       })
       const engineeringUsers = await prisma.user.findMany({
         where: {
           departmentId: engineeringDept?.id,
           isActive: true,
         },
         select: { id: true, name: true, email: true, level: true },
         orderBy: { level: 'desc' },
       })
       ```

    4. Error states:
       - Request not found: notFound()
       - Wrong status: Redirect with message "Request is not ready for solution submission"
       - Not engineering user: redirect('/dashboard')
