<task type="auto">
  <name>Task 1: Add solution approval server actions</name>
  <files>src/server-actions/solutions.ts</files>
  <action>
    Extend `src/server-actions/solutions.ts` with approval actions:

    1. `canUserApproveSolution(solutionId: string)`:
       - Get current user from auth()
       - Check if solution exists and request is in DesignCostEstimationApproval status
       - For custom chains (isCustomChain: true):
         - Find pending approval where requiredApproverId === userId
         - Check all previous orders are approved
       - For hierarchy chains (isCustomChain: false):
         - Find pending approval where requiredLevel === user.level
         - User must be in engineering department
         - Check all previous orders are approved
       - Return { canApprove: boolean, approval?: SolutionApproval, solution?: Solution }

    2. `approveSolution(solutionId: string, comments?: string)`:
       - Validate canUserApproveSolution returns true
       - Use Prisma transaction:
         a. Update SolutionApproval status to 'approved', set approverId, approvedAt
         b. Create RequestActivity (action: 'solution_approved')
         c. Check if this was last pending approval:
            - If yes: update Request status to SendBackToRequester
            - Create notification for requester (type: solution_ready)
         d. If not last: notify next approver in chain
       - revalidatePath('/requests')
       - Return { success: true }

    3. `rejectSolution(solutionId: string, comments: string)`:
       - Validate canUserApproveSolution returns true
       - Validate comments is not empty (required for rejection)
       - Use Prisma transaction:
         a. Update SolutionApproval status to 'rejected', set approverId, comments, approvedAt
         b. Mark all remaining pending approvals as 'rejected'
         c. Create RequestActivity (action: 'solution_rejected')
         d. Update Request status back to SentToEngineer
         e. Create notification for solution submitter
       - revalidatePath('/requests')
       - Return { success: true }

    4. Helper: `notifyNextSolutionApprover(solutionId: string)`:
       - Find next pending approval by order
       - For custom chain: notify specific requiredApproverId
       - For hierarchy: notify all users at requiredLevel in engineering dept
       - Create Notification records

    5. Handle top-level auto-approval in submitSolution (from 04-01):
       - If submitter.level >= maxLevelInEngineering:
         - Create auto-approved SolutionApproval record
         - Set request status directly to SendBackToRequester
         - Skip normal approval chain creation

    Follow patterns from existing `src/server-actions/approvals.ts` for consistency.
  </action>
  <verify>
    - TypeScript compiles: `npx tsc --noEmit`
    - Functions exported: canUserApproveSolution, approveSolution, rejectSolution
    - Rejection requires non-empty comments
    - Status transitions follow three-stage workflow
  </verify>
  <done>
