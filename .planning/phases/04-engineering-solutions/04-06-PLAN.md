<task type="auto">
  <name>Task 1: Add manual completion server action</name>
  <files>src/server-actions/solutions.ts</files>
  <action>
    Add to `src/server-actions/solutions.ts`:

    1. `markRequestComplete(requestId: string, completionNote?: string)`:
       - Validate user is authenticated and has engineering role
       - Validate request exists and status is SendBackToRequester
       - Use Prisma transaction:
         a. Update request status to Completed
         b. Log activity:
            ```typescript
            {
              requestId,
              userId,
              action: 'manually_completed',
              fromStatus: 'SendBackToRequester',
              toStatus: 'Completed',
              comments: completionNote || 'Manually marked as completed by engineering',
            }
            ```
         c. Create notification for requester:
            ```typescript
            {
              userId: request.requesterId,
              type: 'status_changed',
              title: 'Request Completed',
              message: `Your request "${request.title}" has been marked as completed`,
              requestId,
            }
            ```
       - revalidatePath('/requests')
       - Return { success: true }

    2. Error handling:
       - If not engineering user: throw "Only engineering users can mark requests complete"
       - If wrong status: throw "Request can only be marked complete when in SendBackToRequester status"
       - If request not found: throw "Request not found"

    3. Optional: `canMarkComplete(requestId: string)`:
       - Check user is engineering
       - Check status is SendBackToRequester
       - Return { canMark: boolean, reason?: string }
  </action>
  <verify>
    - TypeScript compiles: `npx tsc --noEmit`
    - markRequestComplete exported
    - Status change to Completed works
    - Activity logged correctly
    - Notification created for requester
  </verify>
  <done>
    Manual completion action allows engineering to mark requests as done
  </done>
