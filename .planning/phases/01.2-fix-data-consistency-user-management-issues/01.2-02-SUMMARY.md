## Decisions Made

**Followed plan as specified.** The correct approach for user invitations is to call `createUser()` without the password field, which triggers Clerk's automatic invitation email flow. No separate API calls or manual email sending required.

**Why this works:**
- When `password` field is omitted from `createUser()`, Clerk automatically sends invitation email
- User receives email with link to complete signup and set password
- Simpler than calling separate invitation API
- Standard pattern for admin-created users (documented in Clerk docs)
